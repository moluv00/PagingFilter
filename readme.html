PagingFilter Plugin

by Maurice Wright

Introduction

The HearItLocal Paging Filter plugin allows a developer to create a paged list out of any subset of HTML elements and/or classes. These elements can also be filtered by their class. For example, a <DIV> tag filled with 50 list items <LI> tags as children can have them separated into groups of 3, 5, 20, 22, or whatever count the developer chooses. The plugin has the following basic features:

  • Groups and provides paging for child elements
  • Allows filtering of those child elements by class
  • Automatically creates required navigation based on the number of desired items viewable per page
  • Allows dynamic filtering and associated navigation changes on the client-side
  • It can be used to create paging for multiple sets of data on the same page

Implementation

After installing the required files, the plugin can be attached with the following minimal javascript code. In this case, we're attaching the pagingFilter to an element with an ID of 'performances'.

$(document).ready(function(){
	$("#performances").pagingFilter();                       
});

Requirements

The plugin requires the inclusion of the following:

  • jQuery ( http://www.jquery.com )
  • jquery.pagingFilter.js

Files Needed

After jquery.pagingFilter.js file, the javascript code includes should like something like: 

<script src='https://ajax.googleapis.com/ajax/libs/jquery/1.6/jquery.min.js' type='text/javascript'></script>
<script src="js/jquery.pagingFilter.js" type="text/javascript"></script>

So, the entire javascript will eventually look like:

<script src='https://ajax.googleapis.com/ajax/libs/jquery/1.6/jquery.min.js' type='text/javascript'></script>
<script src="js/jquery.pagingFilter.js" type="text/javascript"></script>
	$(document).ready(function(){
		$("#performances").pagingFilter();
	});
</script>

Filter Links

To create a link that shows a filtered version of our list, we have to make a javacript call to the following function:

$('#performances').pagingFilter.setPagingFilter($('#performances'),1,filter);

What this does is call the 'setPagingFilter' method of the 'pagingFilter' plugin that is attached to the item with an ID of 'performances'. Additionally, the setPagingFilter method takes 3 parameters. First, is the object that it is attached to. In this case $('#performances'). The second parameter is the start page. In this case we're starting with page 1. The last parameter 'filter' is the string value that represents the class you want to filter against. For example, if we want to filter for 'world', we would use the value '.world'. Note, the use of the period prefix that notates that we're filtering for a class called world.

An additional final parameter of 'false' can be added to the 'setPaginFilter' method to prevent the new playlist from automatically playing. It would be set as in the below example code.

$('#performances').pagingFilter.setPagingFilter($('#performances'),1,filter,false);

What we'll do now is create a command in javascript that binds the click events of all anchors within a DIV called 'genre' so that they will perform a filter action against the 'performances' list. The code looks like this:

$('.genre a').live('click', function(e) {
	e.preventDefault();
	var genre = $(this).attr('href');
	$('#performances').pagingFilter.setPagingFilter($('#performances'),1,genre);
});

And, the link code that uses this command might look like this:

<div class="genre">
	<a href="all">All</a>
	<a href=".jazz">Jazz</a>
	<a href=".world">World</a>
	<a href=".soul">Soul</a>
	<a href=".rock">Rock</a>
	<a href=".pop">Pop</a>
	<a href=".jazz.world">Jazz & World</a>
	<a href=".jazz .world">Jazz or World</a>
</div>

The first line of the javascript command looks for all anchor tags that are children of the class 'genre' and does a 'live' bind to the anchor's click event. The second line 'e.preventDefault();' prevents the anchor from performing its normal behavior of trying to direct the page to someplace else. The third line assigns a variable named 'genre' to the class that is passed in through the anchor's HREF parameter. So, obviously, the 'genre', or class that we want to filter against will go in the anchor's HREF parameter.

For example, if we want to see only items in the list with a class of 'jazz', we place the text '.jazz' in the anchor's HREF property. If we want items that have both 'jazz' AND 'world' we would place '.jazz.world' into the HREF parameter. If we want items that have EITHER 'jazz' OR 'world' we would place '.jazz .world' in the HREF parameter, paying close attention to separate the class references with a space.

Page Rotation

In a given collection of items, sometimes its beneficial to rotate through them automatically for the user. This feature can be enabled by providing a time value (in milliseconds) as a 'duration' option. This rotating effect is cancelled when the user clicks anywhere within the area of the element that the plugin is applied to.

By adding a duration of 5 seconds (5000ms) and a transition of 1 second (1000ms) to the first example 

$(document).ready(function(){
	$("#performances3").pagingFilter({
		'itemsPerPage' : 10,
		'duration' : 5000,
		'transition' : 1000,
	});
});

Important Options

PARAMETER DEFAULT VALUE DESCRIPTION SAMPLE VALUES
itemsPerPage 5 The total number of items that should be shown per page. 1,2,...99 etc.
totalPageNumberLinks 3 The total number of linked pages in the navigation. (The current page is not included in that number) 1,2,...99 etc.
childName "div" The class, id, or DOM element that is the child being itemized within the element containing the list items. This will be a prefix to any filter class. "div", "li", ".class-name"
filterName empty A name of a class that the list will be filtered against. Space-separated class names will provide an OR search result. ".soul", ".soul.funk", ".soul .funk", "li"
duration 0 The time in microseconds before the list is automatically navigated to the next page. A value of zero (default) turns off this functionality. 100, 200, 10000, etc.
transition 0 The length of time it takes to transition to a new page when the pages are rotating automatically. A value of zero (default) will use jQuery's default 'slow' value. 100, 200, 10000, etc.
